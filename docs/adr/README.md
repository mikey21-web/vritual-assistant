# Architecture Decision Records

This folder contains Architecture Decision Records (ADRs) for significant architectural decisions made in this project.

## What is an ADR?

An ADR documents a significant decision, including the context, options considered, and the final decision. ADRs help new team members understand why things are the way they are.

## Template

Each ADR follows this format:

```markdown
# ADR-NNN: Title

**Status:** [Proposed | Accepted | Deprecated | Superseded]
**Date:** YYYY-MM-DD

## Context
What's the problem? What constraints exist? What options were considered?

## Decision
What was decided and why?

## Consequences
What trade-offs were accepted? What does this mean for the future?

## Compliance
How will we verify this decision is followed?
```
